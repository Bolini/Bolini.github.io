UX Case Study

In this UX case study, a mobile application experience was designed by framing the problem and establishing a strategy to solve the problem. The design process was explained through methods and choices, as well as the UX and UI of the solution. Following, the case presentation is divided by project description, the user and user needs, process, goals for the design, wireframes, typography and color, and Hi-Fi  

PROJECT DESCRIPTION

Homegy wants to be seen as a company at the forefront of sustainability. They want to help users monitor their appliances' energy use, with automated alerts, and make the smart home scheduled to the user’s life.

THE USER AND USER NEEDS

A targeted user was brought from previous research. Some primary user needs were singled out based on user comments.

  • • 60% male 40% female
  • • Most popular among ages 30-50
  • • Home owners and renters

PROCESS

The user needs were then used to generate possible solutions through a bit of brainstorm to determining goals, and later user flows.

GOALS FOR THE DESIGN

The applications most prominent task is to provide users with tools to live more sustainable and therefore save money, while being accessable and fun to use.
